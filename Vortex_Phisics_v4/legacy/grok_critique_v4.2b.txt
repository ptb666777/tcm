The core math issue (the one that survived scaling)
Right now the force on every pin is identical:
Pythonforce = [-gx, -gy, -gz]   # ← this is -∇Ψ, same vector for +ω and -ω pins
Ψ is built with the signs (weights = rho * omega), so negatives create deep wells and positives create hills.
But because the sensing step never multiplies by the pin’s own omega, every pin (regardless of sign) simply “falls downhill” in Ψ.
→ All pins rush toward negative wells and flee positive hills.
→ Same-sign negatives therefore all feel the exact same force, pile on top of each other, and deepen their own well → total collapse (what you see).
→ Same-sign positives repel each other (good) but are also pulled toward the negative super-nucleus.
This is why the “gravity scaling issue” you saw last time never fully went away — the interaction is still asymmetric. It behaves like “signed sources, unsigned test particles” instead of real charge.
The one-line math fix (this should finally give you symmetric EM-like behavior)
In step_pins, change the force line to:
Pythonforce = p['omega'] * np.array([
    -float(gx[ix, iy, iz]),
    -float(gy[ix, iy, iz]),
    -float(gz[ix, iy, iz]),
])
That’s it.
Now every pin feels own_omega × (-∇Ψ).

Positive pin → attracted to negative wells, repelled by positive hills.
Negative pin → attracted to positive hills, repelled by negative wells.

Result: true opposite-sign attraction + same-sign repulsion, exactly like classical charges (or vortices of opposite circulation).
No more universal collapse of one sign. The 10 negatives will now push each other apart while still being pulled as a group toward the positives → you should see stable or oscillating bound clusters that actually look atomic/molecular instead of one side turning into a black hole.
